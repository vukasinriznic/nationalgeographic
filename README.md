## 📋 Overview

This project is a faithful front-end website of the National Geographic. The goal was to practice advanced HTML/CSS layout techniques and JavaScript interactivity by reverse-engineering a real-world, production-grade website — without using any frameworks or libraries.

---

## ✨ Features

- Pixel-perfect layout matching the original site
- Fully responsive design (desktop & mobile)
- Navigation bar with dropdowns
- Hero section with large imagery and overlaid text
- Article/card grid layout
- Interactive elements powered by vanilla JavaScript
- Consistent typography and color scheme replicating the original

---

## 🧱 Tech Stack

| Technology | Purpose |
|---|---|
| HTML5 | Semantic structure and layout |
| CSS3 | Styling, Flexbox, Grid, animations |
| JavaScript (ES6+) | Interactivity, DOM manipulation |

> **No frameworks. No libraries. Pure front-end fundamentals.**

---

## 🎯 What I Learned

- Advanced **CSS Grid** and **Flexbox** layouts for complex, multi-column designs
- Matching real-world design systems using only browser dev tools
- Writing clean, semantic HTML
- Structuring vanilla JavaScript without frameworks
- Responsive design with media queries

## 🚀 Getting Started

No build tools required. Just open in a browser:

```bash
# Clone the repository
git clone https://github.com/vukasinriznic/natgeo-clone.git
cd natgeo-clone

# Open in browser
open index.html
```

Or simply open `index.html` in your browser directly.
